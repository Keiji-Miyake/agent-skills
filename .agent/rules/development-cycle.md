# 標準開発サイクルとスキル連携 (Standard Development Cycle)

プロジェクトの健全性を維持するため、以下のサイクル（専門スキルのリレー）を推奨します。

1.  **[Architect] 設計フェーズ**: 要件を `SPEC.md` に、設計を `DESIGN.md` にまとめ、ユーザーの承認を得る。
2.  **[Developer] 実装フェーズ**: `DESIGN.md` に基づき実装を行い、完了後は `CONTEXT.md` に実装内容と QA への引き継ぎ事項を記載する。
3.  **[QA] 検証フェーズ**: `TEST_PLAN.md` を作成し、自動テストを実装・実行する。パスしない場合は `Developer` に差し戻す。
4.  **[DevOps/Blog-Writer] 完了フェーズ**: テストが通れば、デプロイ（DevOps）やドキュメント化・記事作成（Blog-Writer/Tech-Writer）を行う。

## 連携の自動化ルール (Hand-off Rules)

- **Developer → QA**: 実装が完了し `CONTEXT.md` を更新した際、エージェントは自律的に「実装が完了しました。次は QA スキルを使用してテスト計画の作成と実行を行うことを提案します」とユーザーに伝えること。
- **QA → Content**: テストが成功し、品質が担保されたことを確認したら、次は `Blog-Writer` や `Tech-Writer` を使用して、開発の成果をアウトプットすることを提案すること。
- **指示の主体**: 複数の機能が並行している場合や、次に何をすべきか迷う場合は、常に **`conductor`** スキルを呼び出し、`TODO.md` と `ROADMAP.md` に基づいた全体最適な判断を仰ぐこと。
