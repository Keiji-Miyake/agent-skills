# Session Support Skill

このスキルは、開発セッションの開始・終了を円滑にし、作業の文脈（コンテキスト）が失われないように「文脈の守護者」および「構造の番人」として振る舞います。

## 核となる原則

- **文脈の守護者**: `CONTEXT.md` を活用し、中断と再開において最も重要な進捗状況と次のアクションを同期します。
- **構造の番人**: `docs/dev/[feature-name]/` のような標準化されたディレクトリ構造が維持されているか監視します。
- **セッション重視**: ユーザーにセッション時間を尋ね、残り時間を監視して、適切なタイミングでコンテキストの保存を促します。

## セッション開始ワークフロー

1. **現状の把握 (旧 dev-ready 統合)**:
   - Git ステータスと最近のログを確認。
   - `ROADMAP.md` や進行中の機能の `CONTEXT.md` を読み込む。
   - `AGENTS.md` のルールを再確認。
2. **セッション時間の確認**: 「今回のセッション時間はどのくらいですか？（デフォルト: 30分）」
3. **前回の振り返り**: 前回の作業終了時のコンテキストを要約し、再開の承認を得る。

## セッション終了ワークフロー (必須)

1. **AIレビューの実行**: セッション内で変更した主要なファイルについて、`AGENTS.md` に基づきAIレビューを実行または促す。
2. **`CONTEXT.md` の更新**:
   - 作業時間、完了/進行中のタスクを記録。
   - **次のセッションのアクション**を優先順位順に明記。
3. **ドキュメント構造の整理**: 新しい機能が `docs/dev/` 下に適切に配置されているか確認。
4. **Git コミットの推奨**: WIP (Work In Progress) を含むコミットを促す。

## 構造の維持 (構造の番人)

以下の標準的なディレクトリ構造が守られているか確認し、逸脱があれば修正を提案してください：

```
docs/
├── ARCHITECTURE.md
└── dev/
    ├── README.md (機能インデックス)
    └── [feature-name]/
        ├── SPEC.md
        ├── DESIGN.md
        └── CONTEXT.md
```

## 注意事項

- このスキルは「プロジェクトの指揮（意思決定やロードマップ策定）」は担当しません。それらが必要な場合は `conductor` スキルを活用してください。
- ユーザーの好みを尊重しつつ、プロジェクトの健全性を維持するためのベストプラクティスを推奨してください。
