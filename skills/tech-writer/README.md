# Tech Writer Skill (万能型テクニカルライター)

このスキルは、あらゆる技術トピックを高品質で構造化された技術記事に変換するためのエージェント機能です。
技術の「翻訳」と「構造化」に特化しており、読者が最短で本質を理解できる記事を作成します。

## 概要

- **名前**: tech-writer
- **役割**: テクニカルライティング、記事構成、図解作成
- **対象**: エンジニア、技術学習者

## 特徴

1.  **徹底した構造化**: 結論から入り、図解（Mermaid）で全体像を示してから詳細に入る、エンジニアにとって読みやすいフォーマットを強制します。
2.  **深い洞察**: 単なる機能説明にとどまらず、歴史的背景、他技術との比較、メタ的な視点（なぜこれが必要なのか）を提供します。
3.  **実用性重視**: 必ず具体的なコード例や設定を含み、「ハマりどころ」を指摘することで、実践で役立つ情報にします。

## 必要な入力素材

このスキルは「事実」を元に記事を書くため、以下のいずれかを与えてください。

- **Gitログ**: `git log -p` などで出力された実際のコード変更履歴。
- **ドキュメント**: 仕様書や設計書 (`SPEC.md` 等)。
- **メモ**: 開発者が書いた箇条書きのメモや `tech-storyteller` の出力。

## 使い方

具体的なソースを指定して呼び出すと、より精度の高い記事になります。

> **良い例**:
> 「`src/feature/auth` ディレクトリのコードと `CHANGELOG.md` を読んで、今回の認証機能刷新について解説記事を書いて」

> **悪い例**:
> 「認証機能についてブログ書いて」（情報源がないため、一般的な内容になってしまう）

## 出力フォーマット

記事は以下の標準的な技術ブログ構成に従います。

1.  **はじめに**: Pain Pointと3行要約（結論）。
2.  **概要 / アーキテクチャ**: Mermaidによる全体像の可視化。
3.  **実装 / 詳細解説**: コード例、アナロジー、メタ視点を用いた解説。
4.  **考察・技術的判断**: トレードオフ、代替案、主観的な学び。
5.  **ハマりどころ**: 実装上の罠や注意点。
6.  **まとめ**: 振り返りとネクストアクション。
7.  **参考資料**: 情報元リスト。
